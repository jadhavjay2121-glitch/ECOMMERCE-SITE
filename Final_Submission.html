2. End-User Documentation: Comprehensive Module Walkthrough

2.1 Category Management Module

The Category Management Module enables administrators to organize products into distinct hierarchies. It provides functionality to view, create, edit, and safely deactivate categories while keeping track of current product allocations securely.

2.2 Product Management Module

The Product Management Module is essential for managing the e-commerce platform's active inventory. It allows administrators to add, modify, and deactivate products while keeping real-time track of inventory stock levels, SKUs, and associated pricing structures.

2.3 Order Management Module

The Order Management Module facilitates the processing and tracking of customer orders. It enables administrators to view all orders natively tied to customers, filter orders smoothly via real-time status drop-downs, and securely update order pipelines (from Pending -> Shipped -> Delivered).

2.4 Customer Management Module

The Customer Management Module is critical for managing user data. It securely tracks personal information (Names, Contacts, Emails) and allows administrators to directly view the aggregated sum of an individual's total order history. It includes rigorous tools for modifying contact data and safely soft-deleting/deactivating idle accounts without compromising historical sales reporting.

2.5 Payment Management Module

The Payment Management Module handles explicit transaction handling mapped natively to system Orders. It acts as an integration interface to view external gateway status. The dashboard clearly logs payment methods (Credit/Debit/PayPal), their processed statuses, and includes a strict one-way trigger for issuing manual Refunds.

2.6 Cart Management Module

The Cart Management Module allows customers to securely queue products with real-time price processing. For administrators, it offers an exclusive Cart Dashboard to analyze cart abandonment rates showing precisely who is holding what products, the exact calculated totals, and enabling manual cart overrides when evaluating inventory reserves.

2.7 Wishlist Management Module

The Wishlist Management Module empowers customers to bookmark high-end items displaying explicit interest without checking out immediately. It aggregates explicit interest pipelines, displaying exactly which product limits are being stalked. It supports single-click triggers shifting items instantly away from being merely Saved into immediately active system Carts for final purchase orchestration if their actual Stock allows.

2.8 Shipping Management Module

The Shipping Management Module fully addresses explicit Logistics pipelines. Once an Order validates, administrators can attach detailed carrier profiles (Courier Names + Web Tracking ID chains) to those records. It accurately models physical package status updates ("Pending", "Shipped", "In Transit", "Delivered") enabling final delivery receipt confirmation within the native app.

2.9 Key Interface Features (Across all 8 Dashboards)

A. Viewing Complex Data Arrays

  • Administrators can navigate seamlessly using the unified left-hand sidebar to jump between all 8 fully-functional dashboards.
  • All grid views natively display item statistics with specialized visual badges. These dynamically capture Boolean database triggers (Active/Inactive), warn about critical inventory (triggering low-stock warnings at <= 5), and showcase dynamic order delivery cycles.

B. Adding and Modifying Entities

  • Administrators can use top-right "Add" buttons to create new entities.
  • Input validation handles mapping associations correctly (e.g., assigning a product to an existing active category via foreign key category_id association).
  • Admins can actively modify pricing (up to 2 decimal precision), descriptions, SKU codes, and inventory stock counts securely via the Edit panel.

C. Data Integrity and Soft Deletions

To safely maintain historical sales logic, the system utilizes rigorous Soft Deletes across both products and categories:

  1. Clicking the red "Deactivate" button triggers a browser warning notifying the admin that removing the item isolates it from the active customer view.
  2. Upon clearing the product's active status, the Boolean triggers to false in the database.
Constraint Safety Feature:
The Category Module is wired to cross-verify the Product database table! The system actively prevents an admin from deactivating a category if there are active products still tied to it. An error banner will prompt the admin to re-distribute those products prior to dropping the category.

3. Project Structure & Setup Requirements

Technology Stack Implemented

  • Back-End MVC Framework: Java 17, Spring Boot 3 (Data JPA / Hibernate)
  • Front-End Presentation Layer: Server-side rendering via Thymeleaf paired with Bootstrap 5 templates.
  • Database: H2 Embedded Engine mapped for dynamic table auto-generation (Configurations pre-set for production MySQL swapping).

Database Design Architecture Achieved

Built fully matching all 8 structured requirements seamlessly:

  • Mapped all Primary Keys (PK) and auto-increment indices using @GeneratedValue(strategy = GenerationType.IDENTITY).
  • prices, amounts, and shipping_cost mapped precisely via decimal(10,2) database formats allowing massive transaction payloads without data overflow.
  • status fields successfully bound to global Boolean definitions allowing for universal soft-deletions on records avoiding catastrophic mapping losses.
  • Deep Entity Binding established across domains. Using @OneToMany and @ManyToOne mapping logic, a single user_id cleanly orchestrates multiple Orders. A single order_id specifically connects explicit transaction records on both the Payment and Shipping tables. Finally, carts / wishlists utilize complex composite FK fetching over both customer_id and product_id efficiently ensuring high-integrity relational links down the entire tree exactly matching all image specs!
